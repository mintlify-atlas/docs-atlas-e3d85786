---

The `/api/admin` route group powers the Eco-It back-office dashboard. Every endpoint in this group is protected by JWT authentication (`verificarToken`) and requires at minimum the `admin` role. Certain destructive operations — such as deleting notifications, purging audit logs, or changing a user's role — additionally require the `superadmin` role. All requests must include a valid `Authorization: Bearer <token>` header.

<Warning>
  Never expose admin tokens in client-side code or version-control repositories. Rotate credentials immediately if a token is compromised.
</Warning>

## Users

### GET /api/admin/usuarios

Returns a paginated-ready list of every registered user, sorted newest-first. Passwords, reset tokens, and expiry dates are excluded from the response. Each user object is augmented with a live `isOnline` flag derived from the active Socket.IO connection map.

**Auth:** Admin

```http
GET /api/admin/usuarios
Authorization: Bearer <token>
```

<ResponseField name="success" type="boolean">
  `true` on a successful fetch.
</ResponseField>

<ResponseField name="usuarios" type="array">
  Array of user objects. Sensitive fields (`password`, `resetPasswordToken`, `resetPasswordExpires`) are stripped.

  <Expandable title="User object fields">
    <ResponseField name="_id" type="string">MongoDB ObjectId of the user.</ResponseField>
    <ResponseField name="nombre" type="string">First name.</ResponseField>
    <ResponseField name="apellido" type="string">Last name.</ResponseField>
    <ResponseField name="email" type="string">Email address.</ResponseField>
    <ResponseField name="rol" type="string">Role: `user`, `admin`, or `superadmin`.</ResponseField>
    <ResponseField name="status" type="string">`active` or `banned`.</ResponseField>
    <ResponseField name="banHasta" type="string | null">ISO date until which the user is banned, or `null`.</ResponseField>
    <ResponseField name="banReason" type="string | null">Human-readable ban reason.</ResponseField>
    <ResponseField name="puntos" type="number">Accumulated eco-points.</ResponseField>
    <ResponseField name="isOnline" type="boolean">Whether the user has an active Socket.IO connection right now.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 registration timestamp.</ResponseField>
  </Expandable>
</ResponseField>

### PATCH /api/admin/users/:id/ban

Bans a user for a specified number of days. Sets the user's `status` to `"banned"`, records `banHasta` (expiry date) and `banReason`, emits a real-time `user:banned` Socket.IO event to the affected user's active connections, and creates a Notification document visible to all admins.

**Auth:** Admin

```http
PATCH /api/admin/users/:id/ban
Authorization: Bearer <token>
Content-Type: application/json
```

| Path parameter | Type   | Description            |
| -------------- | ------ | ---------------------- |
| `id`           | string | MongoDB ObjectId of the target user. |

<ParamField body="dias" type="number" default="7">
  Duration of the ban in days.
</ParamField>

<ParamField body="motivo" type="string" default="Incumplimiento de las normas">
  Human-readable reason for the ban, stored in `banReason` and the generated notification.
</ParamField>

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="message" type="string">
  E.g. `"Usuario baneado por 7 días"`.
</ResponseField>

<ResponseField name="usuario" type="object">
  The updated user document reflecting the new banned status.
</ResponseField>

<Note>
  An admin cannot ban themselves. Attempting to do so returns HTTP 400. A real-time Socket.IO event (`user:banned`) is emitted to the target user's open sockets so they are immediately logged out on the client side.
</Note>

### PATCH /api/admin/users/:id/role

Changes a user's role to either `"user"` or `"admin"`. If the target user is currently `"active"` and connected, their Socket.IO room membership is updated instantly (joined or removed from the `admins` room).

**Auth:** Superadmin

```http
PATCH /api/admin/users/:id/role
Authorization: Bearer <token>
Content-Type: application/json
```

| Path parameter | Type   | Description            |
| -------------- | ------ | ---------------------- |
| `id`           | string | MongoDB ObjectId of the target user. |

<ParamField body="rol" type="string" required>
  New role to assign. Must be either `"user"` or `"admin"`. The value `"superadmin"` is explicitly rejected.
</ParamField>

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="mensaje" type="string">
  Confirmation message including the new role, e.g. `"El rol del usuario ha sido cambiado a admin"`.
</ResponseField>

<ResponseField name="usuario" type="object">
  The updated user document.
</ResponseField>

<Warning>
  Only **superadmins** may call this endpoint. Assigning the `"superadmin"` role via the API is blocked by design — there can only be one superadmin, set directly in the database.
</Warning>

## Notifications

Notifications are system-generated events (e.g. ban events, content alerts) stored in MongoDB. They auto-expire after **7 days** via a TTL index. Each notification tracks which admin users have read it via the `readBy` array.

### GET /api/admin/notifications

Returns the 50 most recent notifications, sorted by date descending. Each notification is augmented with a per-requesting-admin `read` boolean.

**Auth:** Admin

```http
GET /api/admin/notifications
Authorization: Bearer <token>
```

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="notifications" type="array">
  Array of notification objects.

  <Expandable title="Notification object fields">
    <ResponseField name="id" type="string">Alias for `_id`.</ResponseField>
    <ResponseField name="type" type="string">One of `"alerta_obscena"`, `"usuario_baneado"`, or `"alerta_lenguaje"`.</ResponseField>
    <ResponseField name="email" type="string">Email of the user involved.</ResponseField>
    <ResponseField name="nombre" type="string">Full name of the user involved.</ResponseField>
    <ResponseField name="adminName" type="string">Name of the admin who triggered the event.</ResponseField>
    <ResponseField name="dias" type="number">Ban duration in days (for `usuario_baneado` type).</ResponseField>
    <ResponseField name="mensaje" type="string">Human-readable notification message.</ResponseField>
    <ResponseField name="fecha" type="string">ISO 8601 timestamp when the notification was created.</ResponseField>
    <ResponseField name="read" type="boolean">Whether the requesting admin has already read this notification.</ResponseField>
    <ResponseField name="readBy" type="array">Array of admin ObjectIds who have read the notification.</ResponseField>
  </Expandable>
</ResponseField>

## Audit Logs

Audit logs are immutable records of significant admin actions (bans, unbans, role changes, deletions, slide creations, etc.). They are written automatically by the `createAuditLog` utility and **auto-expire after 7 days** via a MongoDB TTL index.

### GET /api/admin/audit

Returns the 50 most recent audit log entries, sorted newest-first.

**Auth:** Admin

```http
GET /api/admin/audit
Authorization: Bearer <token>
```

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="logs" type="array">
  Array of audit log objects.

  <Expandable title="AuditLog object fields">
    <ResponseField name="_id" type="string">MongoDB ObjectId.</ResponseField>
    <ResponseField name="type" type="string">Category of the action, e.g. `"ban"`, `"unban"`, `"delete"`, `"role_change"`, `"slide"`.</ResponseField>
    <ResponseField name="action" type="string">Short label, e.g. `"Usuario Baneado"`, `"Cambio de Rol"`.</ResponseField>
    <ResponseField name="details" type="string">Full description of what happened, including affected email and parameters.</ResponseField>
    <ResponseField name="user" type="string">Name or email of the admin who performed the action. Defaults to `"Sistema"` for automated entries.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 creation timestamp. Records expire 7 days after this date.</ResponseField>
  </Expandable>
</ResponseField>

<Note>
  Audit logs are read-only by design for regular admins. Only superadmins can delete entries, and even then, the TTL index will automatically clean up records older than 7 days regardless.
</Note>

## Map Points (Admin)

These endpoints manage the recycling point markers displayed on the public Eco-It map. Mutations emit a real-time `map:updated` Socket.IO event to all connected clients so the map refreshes instantly without requiring a page reload. Images are uploaded to and hosted on Cloudinary under the `ecoit_map_points` folder.

### GET /api/admin/map/points

Returns all recycling points (both active and inactive, both user-visible and hidden), sorted newest-first. This is the admin-facing counterpart of the public map endpoint.

**Auth:** Admin

```http
GET /api/admin/map/points
Authorization: Bearer <token>
```

<ResponseField name="success" type="boolean">
  `true` on success.
</ResponseField>

<ResponseField name="puntos" type="array">
  Full PuntoReciclaje documents including admin-only fields like `activo` and `visibleToUser`.

  <Expandable title="PuntoReciclaje object fields">
    <ResponseField name="_id" type="string">MongoDB ObjectId.</ResponseField>
    <ResponseField name="nombre" type="string">Display name of the recycling point.</ResponseField>
    <ResponseField name="tipo" type="string">Type: `"recycling"`, `"ecobottle"`, `"truck"`, `"container"`, or `"green_zone"`.</ResponseField>
    <ResponseField name="lat" type="number">Latitude coordinate.</ResponseField>
    <ResponseField name="lng" type="number">Longitude coordinate.</ResponseField>
    <ResponseField name="descripcion" type="string">Optional description.</ResponseField>
    <ResponseField name="imagen" type="string">Cloudinary-hosted image URL.</ResponseField>
    <ResponseField name="activo" type="boolean">Whether the point is currently active.</ResponseField>
    <ResponseField name="visibleToUser" type="boolean">Whether regular users can see this point on the public map.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 creation timestamp.</ResponseField>
    <ResponseField name="updatedAt" type="string">ISO 8601 last-updated timestamp.</ResponseField>
  </Expandable>
</ResponseField>

---

### POST /api/admin/map/points

Creates a new recycling map point. If the `imagen` field contains a base64 data URI, it is uploaded to Cloudinary and the resulting secure URL is stored in its place.

**Auth:** Admin

```http
POST /api/admin/map/points
Authorization: Bearer <token>
Content-Type: application/json
```

<ParamField body="nombre" type="string" required>
  Display name of the recycling point shown on the map and in listings.
</ParamField>

<ParamField body="tipo" type="string" required>
  Category of the point. Must be one of: `"recycling"`, `"ecobottle"`, `"truck"`, `"container"`, `"green_zone"`. Defaults to `"recycling"`.
</ParamField>

<ParamField body="lat" type="number" required>
  Latitude of the point in decimal degrees (e.g. `4.7110` for Bogotá).
</ParamField>

<ParamField body="lng" type="number" required>
  Longitude of the point in decimal degrees (e.g. `-74.0721` for Bogotá).
</ParamField>

<ParamField body="descripcion" type="string">
  Optional text description of the location or accepted materials.
</ParamField>

<ParamField body="imagen" type="string">
  Base64-encoded data URI (e.g. `"data:image/png;base64,..."`) or an existing URL. If a data URI is provided, it is uploaded to Cloudinary and replaced with the resulting secure URL.
</ParamField>

<ParamField body="activo" type="boolean" default="true">
  Whether the point is immediately active after creation.
</ParamField>

<ParamField body="visibleToUser" type="boolean" default="true">
  Whether the point should appear on the public-facing user map.
</ParamField>

**Response — HTTP 201**

<ResponseField name="success" type="boolean">
  `true` on creation.
</ResponseField>

<ResponseField name="punto" type="object">
  The newly created PuntoReciclaje document.
</ResponseField>

<Tip>
  After a successful creation, the server automatically emits a `map:updated` Socket.IO event with the full list of active, user-visible points. Frontend map components should listen for this event to refresh markers without polling.
</Tip>
